# Livox MID360 雷达数据读取指南

## 1. 启动雷达驱动

首先确保雷达驱动正常运行：

```bash
cd ~/ws_livox
source devel/setup.bash
roslaunch livox_ros_driver2 msg_MID360.launch
```

或者使用带RViz可视化的启动文件：

```bash
roslaunch livox_ros_driver2 rviz_MID360.launch
```

## 2. 检查话题信息

### 查看所有话题
```bash
rostopic list
```

### 查看话题类型
```bash
rostopic type /livox/lidar
```

### 查看话题发布频率
```bash
rostopic hz /livox/lidar
```

### 查看话题内容（实时打印）
```bash
rostopic echo /livox/lidar
```

## 3. 数据格式说明

根据launch文件中的 `xfer_format` 参数，数据格式不同：

- **xfer_format=0**: `sensor_msgs/PointCloud2` 格式（标准ROS点云格式）
- **xfer_format=1**: `livox_ros_driver2/CustomMsg` 格式（Livox自定义格式，包含更多信息）

### 话题名称

- **multi_topic=0**（默认）: `/livox/lidar` 和 `/livox/imu`
- **multi_topic=1**: `/livox/lidar_<IP地址>` 和 `/livox/imu_<IP地址>`（每个雷达一个话题）

## 4. 读取数据的方法

### 方法1: 使用Python脚本（推荐）

使用提供的Python脚本：

```bash
cd ~/ws_livox
source devel/setup.bash
python3 read_lidar_data.py
```

脚本会自动订阅 `/livox/lidar` 话题并打印点云数据。

**注意**: 根据launch文件中的 `xfer_format` 参数，可能需要修改脚本中的订阅类型：
- 如果 `xfer_format=0`，使用 `PointCloud2` 订阅
- 如果 `xfer_format=1`，使用 `CustomMsg` 订阅

### 方法2: 使用RViz可视化

```bash
roslaunch livox_ros_driver2 rviz_MID360.launch
```

RViz会自动打开并显示点云数据。

### 方法3: 使用rostopic命令

实时查看点云数据：
```bash
rostopic echo /livox/lidar
```

查看点云统计信息：
```bash
rostopic echo /livox/lidar | grep -E "point_num|points"
```

### 方法4: 使用rosbag录制和回放

录制数据：
```bash
roslaunch livox_ros_driver2 msg_MID360.launch rosbag_enable:=true
```

或者手动录制：
```bash
rosbag record /livox/lidar /livox/imu
```

回放数据：
```bash
rosbag play <bag文件名>
```

### 方法5: 使用C++节点

创建一个C++节点订阅话题：

```cpp
#include <ros/ros.h>
#include <sensor_msgs/PointCloud2.h>
#include <livox_ros_driver2/CustomMsg.h>

void pointcloudCallback(const sensor_msgs::PointCloud2::ConstPtr& msg) {
    // 处理PointCloud2数据
}

void customMsgCallback(const livox_ros_driver2::CustomMsg::ConstPtr& msg) {
    // 处理CustomMsg数据
    ROS_INFO("收到 %lu 个点", msg->point_num);
}

int main(int argc, char** argv) {
    ros::init(argc, argv, "lidar_subscriber");
    ros::NodeHandle nh;
    
    // 根据xfer_format选择订阅类型
    ros::Subscriber sub = nh.subscribe("/livox/lidar", 10, customMsgCallback);
    
    ros::spin();
    return 0;
}
```

## 5. 数据字段说明

### PointCloud2格式字段
- `x, y, z`: 点的3D坐标
- `intensity`: 反射强度
- `tag`: 点标签
- `line`: 扫描线编号
- `timestamp`: 时间戳

### CustomMsg格式字段
- `point_num`: 点的数量
- `lidar_id`: 雷达ID
- `timebase`: 时间基准
- `points[]`: 点数组，每个点包含：
  - `x, y, z`: 3D坐标
  - `reflectivity`: 反射率
  - `tag`: 点标签
  - `line`: 扫描线编号

## 6. 话题延迟创建说明

**重要**：话题是延迟创建的（lazy initialization），只有在雷达开始发送数据时才会创建话题。这意味着：

1. **驱动启动后，话题不会立即出现**
2. **需要等待雷达连接并开始发送数据**（通常几秒到几十秒）
3. **如果雷达未连接或未发送数据，话题就不会出现**

### 检查话题是否出现

```bash
# 持续监控话题
watch -n 1 'rostopic list | grep livox'

# 或者
while true; do
    rostopic list | grep livox && break
    echo "等待话题出现..."
    sleep 1
done
```

## 7. 常见问题

### 问题1: 驱动启动后没有话题出现

这是正常现象！话题是延迟创建的。

**解决方法**：
1. **等待雷达连接**（通常需要5-30秒）
2. **检查雷达是否正常工作**（LED指示灯）
3. **使用诊断脚本**：
   ```bash
   cd ~/ws_livox
   ./check_lidar_connection.sh
   ```
4. **持续监控话题**：
   ```bash
   watch -n 1 'rostopic list'
   ```

### 问题2: 长时间等待后仍没有话题（ROS2）

**现象**：
- 启动后等待超过30秒，`ros2 topic list` 仍然只有 `/parameter_events` 和 `/rosout`
- 驱动日志显示初始化成功，但没有数据发布

**检查清单**：
- [ ] 雷达是否上电（检查LED指示灯）
- [ ] 网络连接是否正常（ping雷达IP）
- [ ] 防火墙是否阻止了UDP端口
- [ ] 配置文件中的IP地址是否正确
- [ ] 雷达和主机是否在同一网段
- [ ] 主机IP地址是否与配置文件中的 `host_ip` 匹配

**详细诊断步骤**：

#### 步骤1: 检查网络连接

```bash
# 1. 检查主机IP地址（应该与配置文件中的 host_ip 匹配）
ip addr show | grep "192.168.1"

# 2. Ping雷达（配置文件中的雷达IP是 192.168.1.157）
ping -c 3 192.168.1.157

# 如果ping不通，检查：
# - 雷达是否上电
# - 网线是否连接
# - 是否在同一网段（192.168.1.x）
```

#### 步骤2: 检查配置文件

```bash
# 查看配置文件
cat ~/ws_livox/src/livox_ros_driver2/config/MID360_config.json

# 确认以下信息：
# - lidar_ip: 应该是雷达的实际IP（默认 192.168.1.157）
# - host_ip: 应该是主机的IP（默认 192.168.1.100）
# - 端口配置是否正确
```

#### 步骤3: 检查端口和防火墙

```bash
# 检查UDP端口是否被占用或阻止
sudo netstat -uln | grep -E "56100|56200|56300|56400|56500"

# 检查防火墙状态（如果使用ufw）
sudo ufw status

# 如果需要，临时关闭防火墙测试（不推荐在生产环境）
# sudo ufw disable
```

#### 步骤4: 检查驱动日志

在启动驱动的终端中，查找以下关键信息：

**正常情况应该看到**：
```
[INFO] Init lds lidar success!
successfully change work mode, handle: 2634131648
successfully enable Livox Lidar imu
livox/imu publish use imu format
livox/lidar publish use livox custom format
```

**如果看到错误信息**：
- `failed to connect` - 网络连接问题
- `timeout` - 雷达响应超时
- `invalid IP` - IP地址配置错误

#### 步骤5: 实时监控话题

```bash
# 在一个新终端中持续监控话题
watch -n 1 'ros2 topic list'

# 或者使用循环检查（带时间戳）
while true; do
    echo "=== $(date +%H:%M:%S) ==="
    ros2 topic list
    echo ""
    sleep 2
done
```

#### 步骤6: 检查雷达状态

```bash
# 检查是否有进程在监听雷达端口
sudo lsof -i :56300 -i :56400

# 检查网络流量（应该能看到UDP数据包）
sudo tcpdump -i eth0 -n udp port 56300 or port 56400
```

#### 步骤7: 验证配置匹配

根据配置文件 `MID360_config.json`：
- **雷达IP**: `192.168.1.157`
- **主机IP**: `192.168.1.100`（必须匹配实际主机IP）

**如果主机IP不匹配**，需要修改配置文件：

```bash
# 查看当前主机IP
ip addr show eth0 | grep "inet "

# 如果主机IP不是 192.168.1.100，需要修改配置文件
# 编辑配置文件
nano ~/ws_livox/src/livox_ros_driver2/config/MID360_config.json

# 将 "host_ip": "192.168.1.100" 改为实际的主机IP
# 然后重新编译
cd ~/ws_livox
colcon build --packages-select livox_ros_driver2
source install/setup.bash
```

#### 步骤8: 重启驱动

```bash
# 完全停止驱动（Ctrl-C）
# 等待几秒后重新启动
cd ~/ws_livox
source install/setup.bash
ros2 launch livox_ros_driver2 msg_MID360_launch.py
```

**常见原因和解决方案**：

1. **IP地址不匹配**：
   - 配置文件中的 `host_ip` 必须与主机实际IP一致
   - 检查：`ip addr show eth0` 应该显示 `192.168.1.100`

2. **雷达未连接**：
   - 检查雷达LED指示灯状态
   - 确保网线连接正常

3. **防火墙阻止**：
   - 临时关闭防火墙测试：`sudo ufw disable`
   - 或添加规则允许UDP端口

4. **雷达未发送数据**：
   - 等待更长时间（有时需要1-2分钟）
   - 检查雷达是否正常工作

5. **ROS2环境问题**：
   - 确保正确source：`source install/setup.bash`
   - 检查ROS2版本：`ros2 --version`

### 问题5: 退出时出现段错误（exit code -11）

**现象**：
- 使用 Ctrl-C 退出驱动时，终端显示 `exit code -11`（段错误 SIGSEGV）
- 出现大量 "DistributeImuData is RequestExit" 和 "DistributePointCloudData is RequestExit" 消息
- 最后显示 "lddc destory" 后进程崩溃

**原因**：
这是 livox_ros_driver2 驱动的一个已知问题，发生在程序退出时的资源清理阶段。通常是由于多线程资源释放时的竞态条件导致的。

**影响**：
- **不影响正常使用**：驱动在运行期间工作正常
- **不影响数据采集**：数据发布和订阅都正常
- **仅影响退出过程**：退出时会出现段错误，但不影响系统稳定性

**解决方案**：

1. **忽略此错误**（推荐）：
   - 这是驱动的一个已知 bug，不影响功能使用
   - 可以安全地忽略这个退出时的错误

2. **使用 kill 命令优雅退出**：
   ```bash
   # 找到进程ID
   ps aux | grep livox_ros_driver2_node
   
   # 使用 SIGTERM 信号退出（比 Ctrl-C 更优雅）
   kill -TERM <进程ID>
   ```

3. **等待自然退出**：
   - 如果可能，让程序自然运行完成，而不是手动中断

4. **使用 gdb 调试**（仅用于开发调试）：
   ```bash
   gdb --args ros2 launch livox_ros_driver2 msg_MID360_launch.py
   # 在 gdb 中运行，可以查看详细的崩溃信息
   ```

**注意**：这个问题已经在 livox_ros_driver2 的 GitHub issue 中被报告，等待官方修复。在修复之前，可以安全地忽略这个退出时的错误。

## 8. 性能优化建议

- 调整 `publish_freq` 参数控制发布频率（默认10Hz）
- 使用 `multi_topic=1` 可以分别处理多个雷达的数据
- 对于实时处理，考虑使用自定义消息格式（CustomMsg）以获得更好的性能
